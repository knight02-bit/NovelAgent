# 任务计划：NovelAgent 框架实现

## 目标
构建一个事件驱动的 AI Agent 框架，用于辅助和主导小说生成。具备知识图谱、多分支时间线、情节规划、多模型头脑风暴等能力，全 GUI 交互。

## 当前阶段
v0 - 核心基础设施

## 各阶段

## 关键问题

1. MemPalace 的 L0-L3 栈如何与系统中 KG 升降级逻辑对接？— 见 findings.md
2. 前端可视化库选型：react-flow vs G6 vs cytoscape 用于 KG 和 DAG？— 需实际对比
3. 多模型脑暴时不同厂商 API 的 error handling 和 fallback 策略？
4. 快照序列化格式（JSON / pickle / protobuf）和存储位置？

## 已做决策

| 决策 | 理由 |
|------|------|
| 架构模式：事件驱动 | 事件是系统的核心通货，天然支持分支和可重放 |
| 存储：MemPalace (ChromaDB + SQLite) | 开箱即用 KG + L0-L3 + 语义搜索，无需额外基础设施 |
| Web：FastAPI + React | 异步原生，图可视化生态丰富 |
| LLM：抽象 Provider + 场景档位 | 解耦厂商，灵活控制成本 |
| 分布式：不做 | 单机异步 IO 足够，数据规模小 |
| Planner 层：Agent Core 之上加一层 | 从"反应式"升级为"目标驱动" |
| 快照：每章结束时自动打 | 避免长事件链全量重放 |
